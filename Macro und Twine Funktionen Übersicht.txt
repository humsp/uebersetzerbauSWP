Macro:
KEINE Verschachtelung!!

	- if/else/endif
		Neues Element im ObjectTree. 
		Ist vom Typ PassageContent. Und hat als Kinder selbst PassagesContens.
	- textinput:
		Neues Element im ObjetTree. 
		Freie Stelle im Text wird z.B. durch "______" umgesetzt.
	- display:
		Neues Element im ObjectTree.
		Ohne überprüfung auf loops.
	- actions:
		Neues Element im ObjectTree. Linkähnlich.
	- choise:
		Neues Element im Objetree. Linkähnlich
	- Button:
		Neues Element im ObjectTree welches von Link erbt
	- radio:
		Umsetzung in ZCode unklar.
	- remember: 
		Eigentliche Funktion wird ignoriert. 
		Soll einfach nur wie set arbeiten.
		forget kann deswegen komplett ignoriert werden.
	- nobr:
		Funktioniert nur für Text der im ObjectTree steht. Das entfernen passiert beim speichern im ObjectTree. D.h: Expressions, display, etc. werden nicht berücksichtigt.
	- silently:
		Wird direkt im Baum ausgewertet. Der Text wird entfernt und alle Prints zu set geändert.

	- checkbox:
		ignorieren, da wir dafür Listen benötigen
